# ByteBites — TF Summary

## TF Notes

The core concept students needed to understand was object-oriented design as a translation process: reading a vague feature request and identifying the real-world "nouns" that become classes, then deciding which attributes and relationships those classes actually need. The most common struggle is with scope because students tend to either under-design (missing key attributes like `popularity_rating`) or over-design by adding fields that feel realistic but were never requested (such as `email`, `password`, or `timestamp`). AI tools like Copilot are most helpful during scaffolding, generating syntactically correct class structures quickly, but they are most misleading at the design stage, where they confidently add plausible-sounding attributes and methods that exceed the specification. A student who accepts AI output uncritically will end up with a system that is more complex than required and harder to test cleanly. To guide a student without giving the answer, ask them to read each attribute or method they wrote and point to the exact sentence in the spec that justifies it. If they cannot find one, that element should be removed.
